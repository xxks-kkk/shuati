1041. Robot Bounded In Circle¶

Note

This Youtube video has a great explanation, which explains the idea behind this LC post.

The central idea is to realize the connection between the impact of instructions on robot and conditions when circle can happen. One whole execution of instructions can impact the robot in two ways:

  1. change the robot position; and

  2. change the direction that the robot is facing.

Robot can be bounded in a circle if after one whole execution of instructions,

  1. the robot stays at \((0,0)\); or

  2. the robot is no longer facing north.

Let’s consider a few examples.

Example 1

Suppose instructions = "GG". Initially, the robot is facing north (indicated by a direction vector \(\langle 0,1 \rangle\)). After two G, the robot moves to the position \((0,2)\) and is still facing north. In this case, no matter how many times instructions are executed, robot is still facing north and circle can never be formed.

Example 2

Suppose instructions = "GGLL". After two G, the robot is at \((0,2)\) facing north. Now, L will make the robot turn left (the direction vector is \(\langle -1,0 \rangle\)) and another L will make the robot turn left again (the direction vector is \(\langle 0,-1 \rangle\)). In this case, one more instructions execution make the robot back to the origin.

Example 3

Suppose instructions = "GGR". After two G, the robot is at \((0,2)\) facing north. Now, R will make the robot turn right (the direction vector is \(\langle 1,0 \rangle\)). The robot follows such instruction will get back to \((0,0)\) after additional three executions as shown in Fig. 1.

Implementation wise, make sure the direction change is calculated correctly. For example, when the robot change from facing north to facing west. The direction vector changes from \(\langle 0,1 \rangle\) to \(\langle -1,0 \rangle\). If the direction vector is \(\langle vec_x, vec_y \rangle\), then after the direction change, the direction vector becomes \(\langle -vec_y, vec_x \rangle\).
