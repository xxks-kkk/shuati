1143. Longest Common Subsequence¶

[Pla19], ref

Main Idea¶

Definition 4 (subsequence)

A subsequence of a sequence \(Z\) is any sequence obtainable from \(Z\) by removing zero or more elements of \(Z\).

Definition 5 (longest common subsequence (LCS))

We are given two sequences \(X = x_1, \dots, x_m\) and \(Y = y_1, \dots, y_n\). A longest common subsequence (LCS) of \(X\) and \(Y\) is a longest sequence that is a subsequence of both \(X\) and \(Y\).

Note

We index elements of a sequence with 1 instead of 0. It won’t make much difference theoretically but it matters for the implementation.

We solve this problem using dynamic programming. For all \(i\) and \(j\) such that \(i \in [0,m]\) and \(j \in [0,n]\), let

\[a(i,j) = \text{the length of an LCS of }X_i = x_1, \dots, x_i~\text{and}~ Y_j = y_1, \dots, y_j\]

There are three possible cases:

  • If \(i = 0\) or \(j = 0\), \(a(0,j) = 0\) and \(a(i,0) = 0\) because elements of a sequence indexed beginning at 0 and when \(i = 0\), that means \(X_i\) is non-existent and thus, by the definition of \(a(i,j)\), \(a(0,j) = 0\). Same argument applies for \(j = 0\).

For the rest two cases, we assume \(i > 0\) and \(j > 0\).

  • If \(x_i = y_j\), then there is an LCS of \(X_i\) and \(Y_j\) that consists of an LCS of \(X_{i-1}\) and \(Y_{j-1}\) followed by the symbol \(x_i = y_j\). Thus, for the length of such LCS, we have \(a(i,j) = a(i-1, j-1) + 1\).

  • If \(x_i \neq y_j\), then an LCS of \(X_i\) and \(Y_j\) is either an LCS of \(X_{i-1}\) and \(Y_j\) or an LCS of \(X_i\) and \(Y_{j-1}\) depending on which one has longer length. So, for the length, we have \(a(i,j) = \max(a(i-1,j), a(i,j-1))\).

In summary, we have

\[\begin{split}\begin{equation} \label{dp-formula} a(i,j) = \begin{cases} 0, & \text{if}~i=0~\text{or}~j=0 \\ a(i-1,j-1) + 1, &\text{if}~i>0~\text{and}~j>0~\text{and}~x_i = y_j \\ \max(a(i-1,j),a(i,j-1)), &\text{if}~i>0~\text{and}~j>0~\text{and}~x_i \neq y_j \end{cases} \end{equation}\end{split}\]

We can compute all the \(a(i,j)\) values in \(O(mn)\) operations. We can trace backwards in the table of \(a(i,j)\) values to obtain an optimal solution in \(O(m+n)\) operations.

Implementation Details¶

Handle Base Case¶

In the above, we consider elements of the sequence with 1-indexed. In actual implementation, the elements of the sequence (e.g., characters of a string) are often 0-indexed. This can cause some trouble in the implementation because we need to handle the special case of \(a(0,j)\) and \(a(i,0)\) as they are no longer 0 for sure; they can be 0 or 1. In other words, \(a(i,j)\) becomes (denoted as \(a'(i,j)\))

\[a'(i,j) = \text{the length of an LCS of }X_i = x_0, \dots, x_i~\text{and}~ Y_j = y_0, \dots, y_j\]

Consider two possible examples:

  • \(X = aaa\) and \(Y = aa\). Then, \(a'(0,j) = 1\) for \(j \in [0,1]\) and \(a'(i,0) = 1\) for \(i \in [0,2]\).

  • \(X = aba\) and \(Y = b\). Then, \(a'(0,0) = 0\) but \(a'(1,0) = 1\) and \(a'(2,0) = 1\).

To avoid special handling for the base case (\(i = 0\) or \(j = 0\)), what we can do is to create a dp table with dimension \((m+1) \times (n+1)\) instead of \(m \times n\). Doing so, we follow \(\eqref{dp-formula}\) precisely and when we compare character, instead of doing text1.charAt(i) == text2.charAt(j), we do text1.charAt(i-1) == text2.charAt(j-1).

Put it differently, to implement \(\eqref{dp-formula}\) precisely, we need to make dp table dimension \((m+1) \times (n+1)\) instead of \(m \times n\).

Variant¶

The same idea is applicable to the problem of longest common substring (LCS). We use the same notation of \(X = x_1, \dots, x_m\) and \(Y = y_1, \dots, y_n\). Then, for all \(i\) and \(j\) such that \(i \in [0,m]\) and \(j \in [0,n]\), let

\[a(i,j) = \text{the length of the common substring that ends at}~x_i~\text{and}~y_j\]

“ends at \(x_i\)” means substring that includes \(x_i\) as the last element.

Note

How \(a(i,j)\) defined here differs from \(a(i,j)\) defined previously.

Then, we have

\[\begin{split}\begin{equation} \label{dp-formula2} a(i,j) = \begin{cases} 0, & \text{if}~i=0~\text{or}~j=0 \\ a(i-1,j-1) + 1, &\text{if}~i>0~\text{and}~j>0~\text{and}~x_i = y_j \\ 0, &\text{if}~i>0~\text{and}~j>0~\text{and}~x_i \neq y_j \end{cases} \end{equation}\end{split}\]

Then, the solution is \(\max(a(i,j))\) for \(i \in [0,m]\) and \(j \in [0,n]\).

To find the actual LCS, what we can do is to note the index of the element of \(X\) (\(Y\) also works) where maximum is reached (denoted \(e\)). Then, we just take a substring \(X[e - max, e]\). See ref for the idea (code itself may not be correct).
