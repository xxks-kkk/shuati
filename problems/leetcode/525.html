525. Contiguous Array¶

Note

  • problem: [Lee]

  • solution

  • ref: [wh0ami]

The key idea of this problem is to use a hashmap \(m\) to keep track of unique sum of numbers that the algorithm has seen so far (denoted as \(s\)) and the associated index of the number that reaches \(s\) for the first time (denoted as \(idx\)). \(m\) then can be used to identify the desired subarray (denoted as \(A\)) as following: algorithm computes accumulated sum of numbers that have seen so far (denoted as \(sum\)). Then, if sum is already in \(m\), then it must be that we have found \(A\) with starting index \(m[sum]\) because each 1 adds one to \(sum\) and each 0 subtracts one from \(sum\). If \(sum\) is found in \(m\), it must be that the same number of 0s and 1s have seen since the value corresponds to \(sum\) first added in \(m\).

Example 7

Consider \(nums = [0,0,1,1,1,0,0]\). The corresponding \(sum\) is \([-1,-2,-1,0,1,0,-1]\). After the first 0, \(m\) contains \(\langle 0,-1 \rangle, \langle -1,0 \rangle\). When algorithm checks first 1, \(sum\) becomes \(-1\) again. The algorithm finds an \(A\), which is \([0,1]\). Intuitively, we can have the following picture:

Let \(i_1\) denote the index of the number that corresponds to \(m[s_1]\). Let \(i_2\) denote the index of the number that \(sum = s_1\). Then, to calculate the length of \(A\), we have \(i_2 - (i_1 + 1) + 1\), which is \(i_2 - i_1\). In this example, \(A = [0,1]\) has length \(2 - (0 + 1) + 1 = 2\).
